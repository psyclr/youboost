## 🏗️ Архитектура

Микросервисная архитектура на Node.js + TypeScript + PostgreSQL + Redis.

**Ключевые принципы:**

- Каждый сервис независим
- API-first подход
- Event-driven communication
- Stateless сервисы для горизонтального масштабирования

## 📦 Микросервисы

### Auth Service

- **Статус**: ✅ Завершен (Phase 2)
- **Тесты**: 150 tests, 98%+ coverage
- **Зависимости**: PostgreSQL, Redis, JWT
- **API**: POST /auth/register, POST /auth/login, POST /auth/refresh, POST /auth/logout, GET /auth/me, POST /auth/verify-email, POST /auth/forgot-password, POST /auth/reset-password

### Billing Service

- **Статус**: ✅ Завершен (Phase 3)
- **Тесты**: 83 tests, 98%+ coverage
- **Зависимости**: Auth Service, PostgreSQL, Crypto payment gateway (stub)
- **API**: GET /billing/balance, POST /billing/deposit, GET /billing/transactions, GET /billing/transactions/:transactionId
- **Internal API**: holdFunds, releaseFunds, chargeFunds, refundFunds (for Order Service)

### Order Service

- **Статус**: ✅ Завершен (Phase 4)
- **Тесты**: 78 tests, 97%+ coverage
- **Зависимости**: Auth Service, Billing Service, Provider Adapter (stub)
- **API**: POST /orders, GET /orders, GET /orders/:orderId, DELETE /orders/:orderId
- **Internal**: Service catalog lookup, price calculation, status transitions
- **Billing Integration**: holdFunds on create, releaseFunds on cancel

### Provider Adapter

- **Статус**: ⚡ Stub реализован (в Order Service)
- **Тесты**: 3 tests (stub)
- **Зависимости**: Нет
- **API**: Internal API для Order Service (ProviderClient interface)
- **Приоритет**: Критический (интеграция с реальными провайдерами)

### Notification Service

- **Статус**: ❌ Не начат
- **Тесты**: 0% coverage
- **Зависимости**: Order Service, Email provider
- **API**: Internal events, webhook callbacks
- **Приоритет**: Высокий (может разрабатываться параллельно)
