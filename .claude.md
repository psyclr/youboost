## 🏗️ Архитектура

Микросервисная архитектура на Node.js + TypeScript + PostgreSQL + Redis.

**Ключевые принципы:**

- Каждый сервис независим
- API-first подход
- Event-driven communication
- Stateless сервисы для горизонтального масштабирования

## 📦 Микросервисы

### Auth Service

- **Статус**: ✅ Завершен (Phase 2)
- **Тесты**: 150 tests, 98%+ coverage
- **Зависимости**: PostgreSQL, Redis, JWT
- **API**: POST /auth/register, POST /auth/login, POST /auth/refresh, POST /auth/logout, GET /auth/me, POST /auth/verify-email, POST /auth/forgot-password, POST /auth/reset-password

### Billing Service

- **Статус**: ✅ Завершен (Phase 3 + Phase 8)
- **Тесты**: 130+ tests, 97%+ coverage
- **Зависимости**: Auth Service, PostgreSQL, Crypto payment gateway (stub)
- **API**: GET /billing/balance, POST /billing/deposit, GET /billing/transactions, GET /billing/transactions/:transactionId, GET /billing/deposits, GET /billing/deposits/:depositId, POST /billing/deposits/:depositId/confirm
- **Internal API**: holdFunds, releaseFunds, chargeFunds, refundFunds, adjustBalance (for Admin)
- **Deposits**: Full deposit lifecycle with Deposit model (PENDING → CONFIRMED/EXPIRED/FAILED), payment gateway integration, wallet credit on confirm

### Order Service

- **Статус**: ✅ Завершен (Phase 4 + Phase 6)
- **Тесты**: 78+ tests, 97%+ coverage
- **Зависимости**: Auth Service, Billing Service, Provider Adapter
- **API**: POST /orders, GET /orders, GET /orders/:orderId, DELETE /orders/:orderId
- **Internal**: Service catalog lookup, price calculation, status transitions
- **Billing Integration**: holdFunds on create, releaseFunds on cancel, chargeFunds/releaseFunds on completion
- **Provider Integration**: selectProvider() for dynamic provider routing
- **Status Polling**: BullMQ background worker polls providers, auto-transitions PROCESSING → COMPLETED/PARTIAL/FAILED
- **Circuit Breaker**: Per-provider circuit breaker skips unhealthy providers (threshold + cooldown)

### Provider Adapter

- **Статус**: ✅ Завершен (Phase 5)
- **Тесты**: 91 tests, 97%+ coverage
- **Зависимости**: Auth Service (admin auth), PostgreSQL
- **API**: POST /providers, GET /providers, GET /providers/:providerId, PUT /providers/:providerId, DELETE /providers/:providerId (admin only)
- **Internal**: selectProvider(), encryptApiKey/decryptApiKey, SMM API HTTP client
- **Features**: AES-256-GCM API key encryption, priority-based provider selection, stub fallback mode

### API Keys Service

- **Статус**: ✅ Завершен (Phase 7)
- **Тесты**: 66 tests, 97%+ coverage
- **Зависимости**: Auth Service, Redis
- **API**: POST /api-keys, GET /api-keys, DELETE /api-keys/:keyId
- **Middleware**: authenticateApiKey (X-API-Key header auth + per-tier rate limiting via Redis)
- **Features**: SHA-256 key hashing, raw key returned once on creation, yb\_ prefix, BASIC/PRO/ENTERPRISE rate limit tiers

### Webhooks Service

- **Статус**: ✅ Завершен (Phase 7)
- **Тесты**: 67 tests, 97%+ coverage
- **Зависимости**: Auth Service, Redis, BullMQ
- **API**: POST /webhooks, GET /webhooks, GET /webhooks/:webhookId, PUT /webhooks/:webhookId, DELETE /webhooks/:webhookId
- **Events**: order.created, order.completed, order.failed, order.partial, order.cancelled
- **Features**: HMAC-SHA256 signatures, BullMQ delivery queue with retry (3 attempts, exponential backoff), fire-and-forget dispatch from order lifecycle

### Notification Service

- **Статус**: ✅ Завершен (Phase 8)
- **Тесты**: 58 tests, 96%+ coverage
- **Зависимости**: Auth Service, Redis, BullMQ, Email provider (stub)
- **API**: GET /notifications, GET /notifications/:notificationId
- **Events**: order.created, order.cancelled, order.completed, order.failed, order.partial
- **Features**: BullMQ delivery queue with retry (3 attempts, exponential backoff), stub email provider (logs to Pino), fire-and-forget dispatch from order lifecycle

### Admin API

- **Статус**: ✅ Завершен (Phase 8)
- **Тесты**: 62 tests, 97%+ coverage
- **Зависимости**: Auth Service (admin auth), All other services
- **API**: GET/PATCH /admin/users, GET/PATCH /admin/orders, POST /admin/orders/:orderId/refund, GET/POST/PATCH/DELETE /admin/services, POST /admin/users/:userId/balance/adjust, GET /admin/dashboard/stats
- **Features**: User management, order management (force status, refund), service catalog CRUD, balance adjustments (ADMIN_ADJUSTMENT ledger), dashboard stats

### Public Catalog

- **Статус**: ✅ Завершен (Phase 8)
- **Тесты**: 60 tests, 97%+ coverage
- **Зависимости**: PostgreSQL, Redis
- **API**: GET /catalog/services, GET /catalog/services/:serviceId (NO auth)
- **Features**: Redis caching (5-min TTL), pagination, platform/type filtering, public unauthenticated endpoints
